# Windows deployment contract

The selected directory must exist, contain at most 4,096 immediate entries, and contain exactly one immediate `.uproject` file. The descriptor is bounded to 1 MiB, must be a JSON object, and may contain only a string `EngineAssociation` of at most 128 characters for discovery purposes. The Engine input defaults to the trimmed `UNREAL_MCP_ENGINE_ROOT` value. A valid configured value remains selected; otherwise registry discovery examines at most 256 Epic Launcher entries and 256 registered user builds. The Engine root must contain `Engine/Build/BatchFiles/RunUAT.bat`, and `Engine/Build/Build.version` must report integer major/minor fields of 5.8 or newer.

The helper calls the repository's fixed `BuildPlugin` source with `-Rocket`, `-NoP4`, `-UTF8Output`, and `-TargetPlatforms=Win64`. It does not use `-Unversioned`, so the result is compatible with the selected Engine build rather than an arbitrary Unreal installation.

Binary installation retains only `Source/UnrealMCP/UnrealMCP.Build.cs` from the module's source tree, because Unreal Build Tool still needs that module rule to consume a precompiled project plugin. The module rule is bounded to 64 KiB. The helper inserts `bUsePrecompiled = true` immediately before the one expected `PCHUsage` assignment and rejects a changed or ambiguous source rule instead of guessing. It removes the module's C++ implementation and headers, every `.dSYM` bundle, and files ending in `.pdb`, `.ipdb`, `.iobj`, `.idb`, `.ilk`, `.obj`, `.pch`, `.map`, or `.debug`, case-insensitively. It retains packaged `Intermediate` precompiled manifests/import libraries needed by Unreal Build Tool. The result must retain an installed `.uplugin` descriptor, the precompiled module rule, and at least one DLL directly under `Binaries/Win64`.

The only deployment destination is `<Project>/Plugins/UnrealMCP`. An existing directory requires explicit replacement authorization. A verified sibling staging directory replaces the old directory by rename; if that rename fails, the old plugin is restored. On success, temporary staging and backup directories are removed.

The displayed LM Studio object has one `mcpServers.unreal-editor` entry. `command` is the absolute Python executable that launched the helper. `args` contains the absolute checkout `server.py` path followed by the exact selected `.uproject` path. Users merge this object into LM Studio's existing `mcp.json`; no token or generated project state is included.
