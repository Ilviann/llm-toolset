# Pin-default and direct-connection contracts

`set_pin_default` carries the common operation/asset/snapshot/graph fields plus one stable `node_id`, stable `pin_id`, and one tagged canonical `default`. The pin must be an unlinked, supported, editable input in the exact live local K2 graph. Execution, wildcard, linked, orphaned, trashed, hidden-default, read-only, ignored, unstable, and reconstructed identities reject before a transaction. `{kind:"engine_default"}` resets the pin to its autogenerated default. Explicit Boolean, numeric, text/name/string/enum, hard/soft object, class, and asset references use the shared K2 codec. Non-default arbitrary structs and live-schema-invalid reference/container forms remain unsupported. Canonical text is limited to 512 characters.

`connect_pins` and `disconnect_pins` carry stable `from_node_id`/`from_pin_id` and `to_node_id`/`to_pin_id` pairs. `from` must be an output, `to` an input, both must belong to the exact graph, and each pin may contain at most 64 links. Connection preflight calls the live K2 schema. Direct make and break-others responses are accepted; break-others replaces only the links named by that schema response. Disallow responses return `incompatible_pins`. Conversion-node and numeric-promotion responses return `conversion_required`, and wildcard pins return `wildcard_specialization_required`; none inserts or reconstructs a node. A duplicate connection and a disconnect for a missing link return `invalid_connection` without a transaction.

Every accepted pin edit uses one transaction, modifies only the owning Blueprint/graph/nodes/pins and schema-selected links, verifies unchanged node/pin object identities, exact default/link postconditions, bounded node count, reciprocal links, and a new authoritative snapshot, then returns the new snapshot and concise changed default or connection record. Unexpected verification failure is undone. Results never include an automatically inserted node. Re-inspect after every edit, compile, Undo/Redo, reload, reconstruction, or bridge restart; a prior pin identity is never retargeted by name.
