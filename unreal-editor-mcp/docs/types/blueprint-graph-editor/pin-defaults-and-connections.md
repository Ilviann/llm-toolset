# Pin-default and direct-connection contracts

`set_pin_default` carries the common operation/asset/snapshot/graph fields plus one stable `node_id`, stable `pin_id`, and one tagged canonical `default`. The pin must be an unlinked, supported, editable input in the exact live local K2 graph. Execution, wildcard, linked, orphaned, trashed, hidden-default, read-only, ignored, unstable, and reconstructed identities reject before a transaction. `{kind:"engine_default"}` resets the pin to its autogenerated default. Explicit Boolean, numeric, text/name/string/enum, hard/soft object, class, and asset references use the shared K2 codec. Non-default arbitrary structs and live-schema-invalid reference/container forms remain unsupported. Canonical text is limited to 512 characters.

`connect_pins` and `disconnect_pins` carry stable `from_node_id`/`from_pin_id` and `to_node_id`/`to_pin_id` pairs. `from` must be an output, `to` an input, both must belong to the exact graph, each pin may contain at most 64 links, and the proposed direction must not form a node cycle. Connection preflight calls the live K2 schema. Direct make and break-others responses are accepted; break-others replaces only the links named by that response. Live wildcard specialization and numeric promotion are also accepted without conversion opt-in. Endpoint pin/node reconstruction is detected by object/type/identity read-back; every live identity on a reconstructed endpoint is returned in `reconstructed_identities` and its complete bounded node record appears in `changed.nodes`.

`connect_pins` alone accepts optional `automatic_conversion`. Omission or `false` returns `conversion_required` without mutation when the schema requires a conversion node. `true` permits only the live schema's conversion response, preflights capacity for the published maximum of one inserted node, and includes that insertion in the same transaction. The inserted node and all pins must have stable GUIDs, stay within the 256-pin/64-link bounds, and form a reciprocal directed path from the requested output to input through only that node. Its node and pin IDs appear in `created_identities`, and its node record appears in `changed.nodes`. Disallowed conversions return `incompatible_pins`; duplicate, missing, cyclic, over-limit, unstable, or stale requests reject before a transaction.

Every accepted pin edit uses one transaction, modifies only the owning Blueprint/graph/nodes/pins, schema-selected links, and any explicitly permitted conversion node, verifies exact default/link/path postconditions, bounded node count, reciprocal links, stable live identities, and a new authoritative snapshot, then returns the new snapshot and a concise operation-discriminated change. Connection records report `connected`, `direct`, `automatic_conversion`, `wildcard_specialized`, `conversion_node_count`, and `replaced_link_count`. Unexpected insertion or verification failure is undone. Re-inspect after every edit, compile, Undo/Redo, reload, reconstruction, or bridge restart; use returned reconstructed identities only to interpret the completed result, then use fresh inspection identities for the next request.
