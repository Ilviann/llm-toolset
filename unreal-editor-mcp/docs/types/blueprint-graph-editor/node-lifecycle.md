# Graph-node lifecycle contracts

`blueprint_graph_edit` performs exactly one `add_node`, `move_node`, or `remove_node` operation. Every shape carries a caller-generated operation ID, exact mutable Actor Blueprint path, current 40-character snapshot, and stable 32-character local graph GUID. `add_node` additionally carries one retained 32-character action ID and `{x,y}` position; move carries one stable node GUID and position; remove carries one stable node GUID. Coordinates are bounded integers within ±1,000,000.

For `add_node`, the action catalog must still retain an identity bound to the same bridge, target generated class, asset, K2 graph/schema, snapshot, normalized filters, and optional pin context. The catalog scans its rebuild signature from the live Blueprint action database within the normal 20,000-spawner/one-second ceiling and reapplies Unreal's live filter. Unknown, expired, evicted, mismatched, or newly filtered identities return `invalid_action` before a transaction. No cached UObject pointer is trusted.

Creation invokes the resolved spawner once. A newly created node and all of its pins must carry persistent Unreal GUIDs; the result lists those `created_identities`. A unique spawner may return an existing node, which is not repositioned and is reported as `created: false` and `returned_existing: true`. Null, out-of-graph, over-limit, unstable, or over-256-pin results are explicitly restored and reject.

Move/remove resolve only by stable node GUID inside the exact graph. Local event graphs, editable user-function graphs, and local macro graphs are supported. Inherited, interface, construction, delegate/signature, intermediate, transient, non-K2, and other read-only graphs reject. Intermediate nodes, required entry/result/tunnel nodes, and any node for which Unreal returns false from `CanUserDeleteNode()` reject as `protected_node`. Removal breaks only the target node's existing links; pin defaults and connection creation remain unreleased.

At most 2,048 nodes may be present in a mutable graph. Every accepted edit uses one Unreal transaction, applies structural/non-structural Blueprint marking according to the K2 node, and returns the asset, operation, graph, new snapshot, dirty state, one complete bounded changed-node record, created/existing flags, and created identities. Postconditions are verified by node identity/position/absence plus authoritative snapshot inspection. Unexpected failure is undone; no-change spawner failure cancels the transaction. Re-inspection is required after every graph edit, compile, Undo/Redo, reload, reconstruction, or bridge restart.
