# Action catalog internal boundaries

`FActionCatalogQuery` owns exact JSON-shape validation and normalized asset, graph, snapshot, text, owner, function, member, family, and limit filters. The facade verifies the authoritative inspection snapshot, resolves the live Blueprint, K2 graph, and optional pin, and binds those identities into the query digest.

`ScanActions` owns exact-loaded-owner-first, target-class-next, then global bounded traversal of `FBlueprintActionDatabase`, live `FBlueprintActionFilter` checks, pre-existing unique-event suppression, supported-family classification, deduplication, deterministic sorting, and bounded candidate record encoding. Classification orders event, cast, flow-control, operator, and literal specializations before generic function and variable spawners. The facade owns expiry, eviction, opaque action IDs, rebuild signatures, bridge/snapshot/schema/pin identity, cached public result reconstruction, and the narrow Phase 11 live-resolution method. Resolution repeats bounded exact-owner/target-first traversal and live filtering and returns one ephemeral spawner/binding pair only to the graph editor on the Game thread.
