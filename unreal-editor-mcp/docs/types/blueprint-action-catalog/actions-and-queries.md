# Action queries and records

`blueprint_action_catalog` accepts exactly `asset_path`, `graph_id`, and `expected_snapshot`, plus optional `text`, `owner_class`, `function`, `member`, `node_family`, `pin_context`, and `limit`. `text` matches the complete menu title or member name case-insensitively. Owner, function, and member filters are exact. Node family is `function_call`, `variable_get`, `variable_set`, `event`, `flow_control`, `cast`, `literal`, or `operator`. Pin context contains one exact stable `node_id` and `pin_id` from the selected graph. The default limit is 20 and the maximum is 50.

The target must resolve to a published Blueprint family and a live graph with the supplied GUID. The inspector recomputes the full structural snapshot before every catalog access; a mismatch returns `stale_precondition`. An unknown graph or pin returns `not_found`. `function` is valid for function-call, event, literal, and operator families; `member` is valid for variable get/set. Conflicting function/member/family filters return `invalid_argument`.

A result contains the bridge instance, exact `blueprint_family`, normalized asset path, graph and snapshot IDs, `actions`, returned/scanned counts, `truncated`, `timed_out`, and the remaining advertised identity lifetime. Each action has an opaque 32-character `action_id`, `node_family`, bounded display title/category, exact owner path and member name, member kind, and `wildcard`. Function-backed actions additionally report pure, static, const, and latent flags; cast actions report `class_cast`.

Events are available only in event graphs and omit event functions already implemented as unique events. Flow control includes branches, sequences, multi-gates, do-once nodes, switches, enum iteration, and live-compatible standard flow-control macros. Cast ownership identifies the target class. Literals include Kismet `MakeLiteral*`, enum, bitmask, and self actions. Operators include promotable wildcard operators and common associative operators. Unreal's live action filter excludes latent calls from incompatible function graphs and applies Blueprint, graph, static/instance, inheritance, and pin-context restrictions to every family.

Action IDs are not signatures and may be supplied only to Phase 11 `blueprint_graph_edit.add_node`. Internally a retained record binds the ID to bridge instance, target generated class, graph schema/GUID, snapshot, normalized query and pin context, plus a rebuildable family/owner/member/action-owner/spawner signature. Invocation scans that signature from the live database and reapplies the exact graph/pin action filter; no cached spawner pointer is trusted. Identical queries reuse live IDs. IDs expire after 60 seconds, and are invalid after record eviction, structural change, or bridge restart. At most 32 catalogs and 256 action records are retained.

The target asset/class hierarchy is scanned before the remaining global database. At most 20,000 spawners are examined for at most one second, and one catalog runs at a time on the Game thread. Result-limit, scan-limit, or elapsed-time termination sets `truncated`; elapsed-time termination also sets `timed_out`. The shared 64 KiB request and 256 KiB response ceilings remain authoritative.
